모델·하네스·인증 해석 + 루프 준비

"이번 턴을 어떤 모델·하네스·인증으로 돌릴지" 확정하고, 어템프트 루프가 쓸 상태와 컨텍스트 엔진을 초기화한다.

무엇을 하나

  • 워크스페이스 · 플러그인 — 워크스페이스 디렉토리 해석(폴백 경고 포함) + 런타임 플러그인 로드.
  • 초기 모델·프로바이더 + 훅 — 초기 모델 해석 후 before_agent_reply 훅 실행(처리된 응답으로 조기 반환 가능), resolveHookModelSelection으로 훅이 모델을 오버라이드할 수도 있다.
  • 하네스 선택 + 풀 모델 해석 — 하네스를 고르고 후보 프로바이더들을 순회하며 모델을 해석(미지 모델이면 FailoverError).
  • 인증 프로필 — 프로필 후보 순서를 계산하고 auth 컨트롤러를 생성·초기화.
  • 루프 준비 — 재시도 캡·idle 타임아웃 브레이커·컴팩션 카운터·usage 누적기 등 런루프 상태를 초기화하고, 컨텍스트 엔진을 한 번 해석해 어템프트 간 재사용.

코드 위치 (검증됨)

  • run.ts:948 워크스페이스 · run.ts:969 ensureRuntimePluginsLoaded
  • run.ts:977–1039 초기 모델 해석 + cron 훅 + resolveHookModelSelection
  • run.ts:1043–1164 하네스 선택 + 모델 해석 루프(미지 시 FailoverError)
  • run.ts:1383–1533 인증 프로필 후보 → createEmbeddedRunAuthController → initializeAuthProfile
  • run.ts:1560–1802 런루프 상태 초기화 + resolveContextEngine

연결 — provider/model 주입 지점

provider/model은 여기(아웃터)에서 해석되어 이너 어템프트로 주입된다. 게이트웨이가 모델을 고르는 게 아니라 이 해석 계층이 고르며, 페일오버가 후보 (provider, model)을 바꿔가며 재시도하는 것도 이 지점을 통해서다.
