프로젝트 디렉토리별 역할 안내서

OpenClaw는 자신의 기기에서 직접 실행하는 개인용 AI 어시스턴트다. WhatsApp · Telegram · Slack · Discord · iMessage 등 이미 쓰는 채널에서 응답하고, 음성으로 말하고 들으며, 라이브 Canvas를 렌더링한다. Gateway는 컨트롤 플레인일 뿐, 제품의 본질은 어시스턴트 그 자체다. 모노레포의 디렉토리 구조와 각 영역의 역할은 다음과 같다.

루트 디렉토리 16+ src/ 모듈 68 extensions/ 플러그인 139 packages/ 라이브러리 21 skills/ 스킬 52 언어 TypeScript · Swift · Kotlin
최상위 구조 src/ 코어 런타임 packages/ extensions/ apps · ui 문서 · 운영 루트 파일

최상위 디렉토리

root/

모노레포 루트의 주요 디렉토리다. 코어 런타임(src)을 중심으로, 재사용 라이브러리(packages)·플러그인(extensions)·네이티브 앱(apps)이 위성처럼 배치되어 있다.

⚙️src/
코어 TypeScript 런타임 — 프로젝트의 심장
Gateway·에이전트·채널·CLL·플러그인 로더·세션·도구 등 OpenClaw의 핵심 로직 전부. 플러그인에 비의존적(plugin-agnostic)이며, 플러그인은 SDK 시암(seam)을 통해서만 코어로 진입한다.
68개 모듈핵심
📦packages/
재사용 가능한 내부 라이브러리 패키지
코어와 플러그인이 공유하는 독립 패키지. gateway-protocol(통신 규약), llm-core·llm-runtime(LLM), agent-core, plugin-sdk 등으로 구성된다.
21개 패키지
🧩extensions/
플러그인 — 프로바이더 · 채널 · 도구
anthropic·openai·amazon-bedrock 등 모델 프로바이더, discord·brave 등 통합 기능. 코어 dist에 번들되는 내부 플러그인과 자체 패키지/의존성을 소유하는 외부 공식 플러그인으로 나뉜다.
139개 플러그인
🛠️skills/
외부 도구 · 서비스 연동 스킬
1password·notion·github·obsidian·peekaboo 등 외부 앱/서비스를 어시스턴트가 다루게 해주는 스킬 모음. 각 스킬은 독립 폴더로 캡슐화된다.
52개 스킬
📱apps/
네이티브 클라이언트 앱
android(Kotlin)·ios·macos(Swift)·macos-mlx-tts(로컬 TTS)·swabble 앱과, 이들이 공유하는 shared/OpenClawKit. 음성·트레이·노드 모드 등 기기별 경험을 담당한다.
Swift · Kotlin
🖥️ui/
웹 기반 컨트롤 UI (Vite)
게이트웨이를 제어하는 웹 프론트엔드. 라이브 Canvas·설정·채팅 화면을 렌더링한다. src·public·vite.config.ts로 구성된 독립 Vite 앱이다.
Vite · TS
📚docs/
문서 소스 → docs.openclaw.ai
설치·채널·플러그인·아키텍처·레퍼런스 문서의 원본. docs-sync-publish.yml을 통해 미러 빌드 후 R2 + Worker 라우터로 발행된다.
발행 소스
✅qa/
QA 시나리오 · 성숙도 평가
테스트 시나리오(scenarios), 성숙도 점수(maturity-scores.yaml), 프런티어 하네스 계획, 자격증명 브로커 등 품질 검증 자산.
🧪test/
통합 · E2E 테스트 하네스
패키지 경계·릴리스·런처·플러그인 import 경계 등을 검증하는 통합/E2E 테스트와 fixtures·helpers·mocks. 단위 테스트는 각 소스 옆 *.test.ts에 위치한다.
📜scripts/
빌드 · 릴리스 · 벤치 · 검사 스크립트
빌드(build-all.mjs)·안드로이드 릴리스·성능 벤치(bench-*)·아키텍처 스멜 검사·변경 레인 분석 등 개발/CI 자동화 스크립트.
🔧config/
공유 빌드 · 린트 설정
tsconfig 베이스, knip(미사용 코드), markdownlint, shellcheck, swiftformat/swiftlint 등 도구 설정의 공통 보관소.
🚀deploy/
배포 환경 설정
프라이빗 배포용 fly.private.toml. 루트의 fly.toml·render.yaml·docker-compose.yml과 함께 호스팅 구성을 이룬다.
🔐security/
보안 스캔 규칙
opengrep 정적 분석 규칙 등 보안 검사 자산. 루트 SECURITY.md·.semgrepignore와 연계된다.
📝.agents/
메인테이너 노트 · 에이전트 스킬
유지보수자용 노트(maintainer-notes)와 내부 자동화 에이전트 스킬(autoreview·clawsweeper·crabbox 등) 정의.
🤖.github/
CI · GitHub 워크플로우
CI 파이프라인, 이슈/PR 템플릿, 자동화 워크플로우 정의.
🪝git-hooks/ · patches/
Git 훅 · 의존성 패치
git-hooks/pre-commit 커밋 전 검사 훅과, npm 의존성을 직접 수정하는 patches/*.patch(예: claude-agent-acp).

extensions/ — 플러그인

139 plugins

코어의 일반 루프 위에 기능을 얹는 플러그인. 모델 프로바이더가 auth/카탈로그/런타임 훅을 소유하고, 메시지 채널은 전송 전용으로 유지된다. 코어 src/**에 직접 의존할 수 없고 공개 배럴만 사용한다.

🤝모델 프로바이더
LLM/AI 백엔드 통합
anthropic·anthropic-vertex·amazon-bedrock·azure-speech·cerebras·cohere·copilot·deepseek·codex 등. 각자 인증·모델 카탈로그·런타임 훅을 소유한다.
💬채널 어댑터
메시징 플랫폼 전송 계층
discord 등 채널 플러그인은 전송 전용 — 이식 가능한 표현/액션을 렌더링하고 전송 한계를 강제하며 콜백 봉투를 매핑한다. 제품 명령 트리나 정책은 소유하지 않는다.
🧰도구 · 통합
검색 · 미디어 · 디바이스 기능
brave·duckduckgo·exa(검색), browser·canvas·comfy·elevenlabs·deepgram(미디어), bonjour·device-pair·diagnostics-otel 등 기능 확장.
📖스코프 가이드
AGENTS.md · CLAUDE.md
extensions/ 루트에는 플러그인 작성 규칙을 담은 AGENTS.md·CLAUDE.md 가이드가 있어, 코어 경계와 SDK 시암 사용 규칙을 안내한다.
