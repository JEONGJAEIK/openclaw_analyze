출구 — 종료 판정 → 결과 반환

루프가 끝나는 곳. 이번 턴이 쓸 만한 답을 냈는지 판정하고, 결과를 만들어 런타임 밖으로 내보낸 뒤 자원을 정리한다.

무엇을 하나 (순서대로)

  • incomplete-turn 분류 — resolveIncompleteTurnPayloadText로 "안전한 최종 답이 있는지" 판정.
  • 종료 outcome — 정상이면 실제 어시스턴트 응답(terminalPayloads)을, 아니면 ⚠️ incomplete_turn 페이로드를 만든다.
  • 결과 반환 — EmbeddedAgentRunResult(payloads + meta + 배달 플래그)를 반환. 이게 런타임의 출구.
  • 정리(finally) — fast-mode 리셋, 프롬프트 캐시 drop, auth refresh 타이머 stop, 컨텍스트 엔진 dispose → 두 레인 큐 unwind.
  • 소비 — auto-reply 계층이 .payloads/.meta를 받아 채널로 전송.

코드 위치

  • run.ts:3794–3814 incomplete-turn 분류 (incomplete-turn.ts:225)
  • run.ts:3845–4001 incomplete terminal · run.ts:4043–4099 정상 terminal
  • run.ts:4146–4191 finally 정리 + 레인 unwind
  • types.ts:188–223 EmbeddedAgentRunResult 형태 (error.kind에 incomplete_turn)
  • 소비처: auto-reply/reply/agent-runner-execution.ts:2506 등

하네스 경계 — 자체도 외부도 같은 계약

여기서 한 가지가 분명해진다. 지금까지 본 정교한 런타임(runEmbeddedAttempt)은 사실 "openclaw"라는 이름의 하네스 하나일 뿐이다. 모든 시도는 먼저 하네스 선택을 거친다.

  • AgentHarness 계약 — 능력들의 합성. 필수는 둘: supports(ctx)(이 실행을 맡을 수 있나? → { supported, priority })와 runAttempt(p)(맡으면 어떻게 실행하나? = 하네스 본체). 선택: compact? · classify? · reset?/dispose?.
  • 자체 하네스 — createOpenClawAgentHarness는 id:"openclaw", runAttempt = runEmbeddedAttempt, priority 0(항상 가능하되 최후의 기본값)인 14줄 어댑터. 자체 런타임을 특권화하지 않는다.
  • 외부 하네스 — Codex(id:"codex")·Copilot(id:"copilot")이 플러그인 SDK api.registerAgentHarness(...)로 동일 계약에 꽂힌다. OpenClaw가 Codex를 "감싼다"기보다, Codex가 OpenClaw의 하네스 슬롯에 꽂힌다.
  • 선택 6가지 이유 — forced_openclaw · forced_plugin · auto_plugin · auto_openclaw · implicit_plugin_unavailable_openclaw · cli_runtime_passthrough_openclaw. 6개 중 4개가 …_openclaw로 끝난다 — 외부가 없거나·안 맞거나·불명확하면 항상 자체로 안전 폴백.
  • harness/types.ts:121–125 계약 · builtin-openclaw.ts:14 id:"openclaw" · selection.ts:325 호출 · backend.ts:10 진입
  • extensions/codex/harness.ts · extensions/copilot/harness.ts 외부 동료

바깥 껍데기는 항상 OpenClaw의 것

OpenClaw로 Codex를 쓰면: 게이트웨이·디스패치, 선제 컴팩션, 페일오버·재생 안전성이라는 운영 껍데기가 Codex 하네스를 감싼다. 자체 런타임을 쓰면: 같은 껍데기가 openclaw 하네스를 감싼다. 외부 하네스는 단일 턴 runAttempt만 끼워질 뿐, 세션·페일오버·컴팩션·종료 판정은 모두 run.ts 소유다.

비법 소스는 추론 하네스를 이기는 게 아니라, 어떤 하네스든 끼울 수 있는 껍데기다. 껍데기는 모델의 사고를 건드리지 않기에 품질을 떨어뜨리지 않고, 신뢰성·편재성·지속성을 더한다. 그래서 "감쌌는데 더 좋다."
