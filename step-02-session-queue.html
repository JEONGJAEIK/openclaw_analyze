세션 해석 + 2단 레인 큐

요청을 올바른 세션으로 묶고, 두 겹의 레인 큐로 동시 실행을 직렬화한다 — 경쟁 상태 없이 한 번에 하나씩.

무엇을 하나

  • 세션키 백필 — 호출자가 키를 생략해도 다운스트림(훅·LCM·컴팩션)이 항상 non-null 키를 받도록 먼저 해석한다.
  • 세션 해석 — sessionId·sessionFile·agentId를 확정하고 파라미터를 재구성.
  • 세션 레인 큐 — enqueueSession으로 같은 세션의 실행을 직렬화(순차 처리).
  • 지연 정비 체크포인트 — 세션 레인 안에서, 보류된 transcript 재작성이 있으면 먼저 완료시킨다.
  • 글로벌 레인 큐 — enqueueGlobal로 교차 세션 동시성을 제한하고 run 컨텍스트/lifecycle 세대를 회수.

코드 위치 (검증됨)

  • run.ts:630 backfillSessionKey() (정의 run.ts:510)
  • run.ts:636–646 resolveAgentRunSessionTarget() + 파라미터 재구성
  • run.ts:647–648 resolveSessionLane/resolveGlobalLane (lanes.ts:6/11)
  • run.ts:786 세션 레인 큐 진입 enqueueSession
  • run.ts:791 지연 turn-maintenance 체크포인트
  • run.ts:793 글로벌 레인 큐 진입 enqueueGlobal

참고 — 레인 큐 vs 파일 락

아웃터의 동시성 제어는 2단 레인 큐다. 단, 헷갈리기 쉬운 점 하나:

  • 여기서의 큐잉(레인)은 실행 단위 직렬화다. 세션 파일 쓰기 락은 별개로, 이너 어템프트(attempt.session-lock.ts)에서 획득한다 — 같은 "락"이라는 단어로 묶지 말 것.
