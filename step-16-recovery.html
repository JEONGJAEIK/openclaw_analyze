복구 — 페일오버 · 컴팩션 · 재생안전

모델은 수십 가지로 실패하고, 긴 세션은 컨텍스트가 넘친다. OpenClaw는 실패 종류마다 다른 복구를 두고, 요약은 최후에·요약마저 감사하며, 부수효과가 있으면 재생을 막는다. 모델의 사고를 건드리지 않고 신뢰성·지속성을 더하는 운영 계층이다.

위치 — 안쪽 복구 vs 바깥쪽 복구

  • 안쪽(스트림 후처리): 도구호출/stop-reason 복구는 모델 출력 스트림을 감싸 한 호출의 출력을 고친다 (디스패치의 래핑 스택).
  • 바깥쪽(루프): 페일오버 사다리·컴팩션·재생안전은 어템프트가 끝난 뒤 run.ts에서 "다시 돌릴지/어떻게"를 결정한다.

실패 → 복구 지도

각 실패 모드에 대응 복구가 따로 있다:

  • ⏳ 단기 레이트리밋 → 같은 모델 재시도(maybeRetrySameModelRateLimit), 길면 프로필/모델로 에스컬레이션.
  • 🔥 과부하(overload) → 백오프 후 프로필 회전(overloadProfileRotationLimit까지), 넘으면 모델 폴백.
  • 🔑 인증/결제 실패 → 프로필 표시 후 다음 프로필로(advanceAuthProfile); 결제는 자격 모드별 안내 분기.
  • 💤 무응답 행(idle hang) → idle 워치독이 끊고 같은 모델 1회 재시도(same_model_idle_timeout).
  • ❓ Unhandled stop reason → 안정적 오류 메시지로 재작성(동기/비동기/iterator/result 4경로).
  • 🫥 추론만/빈 응답 → "이어서" 재시도(추론 ≤2회, 빈 응답 ≤1회).
  • 🧩 망가진 도구 호출 → 인자 파싱 복구(여러 방언).
  • 🛑 복구 불가(한도 소진) → 재생 안전하면 FailoverError로 모델 전환, 아니면 오류 페이로드.

① 페일오버 5액션 사다리 — 가벼운 것부터

실패는 5개 액션 중 하나로 매핑된다. 운영 흐름은 싼 복구 → 비싼 종결로 단조 증가한다 — 한 단계가 풀면 거기서 멈춘다.

  • 1 · continue_normal — 일시적·무시 가능, 그대로 진행 (최저 비용)
  • 2 · retry (same model) — idle/단기 레이트리밋, 모델·프로필 안 바꾸고 1회 (제자리 복구)
  • 3 · rotate_profile — 인증 프로필 회전, 다른 자격증명으로 (계정 전환)
  • 4 · fallback_model — FailoverError 던져 외부 루프가 폴백 모델로 (모델 전환)
  • 5 · surface_error / return_error_payload — 복구 소진, 안정적 오류 표시 (종결)

② 불완전 턴 복구 — "이어서, 재시작 금지"

"끝났지만 답이 없는" 턴을 종류별로 다르게 재시도한다:

  • 추론만 한 턴: 최대 2회 — "추론은 기록했지만 보이는 답이 없다. 그 부분 턴에서 이어서 답을 내라. 반복·재시작 금지."
  • 빈 응답: 최대 1회 — 마찬가지로 "현재 상태에서 이어서, 처음부터 재시작 금지".
  • "이어서/재시작 금지" 명시가 모델이 같은 작업을 반복하거나 루프에 빠지는 걸 막고 부분 진행을 살린다.
  • incomplete-turn.ts:133 분류·재시도 지시 (종료 판정은 출구에서)

③ 선제적 컴팩션 — 보내기 전에 압력을 잰다 + 4-루트

넘치기 전에 실측 기반으로 압력을 재고, 곧장 요약하지 않고 손실 최소 루트를 고른다.

  • 실측 우선: 추정이 아닌 렌더된 프로바이더 경계의 실제 토큰 수를 쓴다.
  • 과소평가 방지: 윈도잉된 것과 안 된(unwindowed) 전체를 모두 추정해 더 큰 값 채택.
  • 프롬프트 몫 사수: 예약(reserveTokens)이 커도 사용자 프롬프트 최소 예산(minPromptBudget)은 남긴다.

4-루트 결정 — 원칙: 대화 추론을 손상시키는 요약보다, 부피 큰 도구 출력의 절단을 우선:

  • fits — 넘침 없음, 그대로 (손실 0)
  • truncate_tool_results_only — 줄일 도구 결과가 넘침을 넉넉히 초과(임계 = max(overflow+512t, overflow×1.5)), 요약 없이 도구 출력만 절단 (대화 무손실)
  • compact_then_truncate — 절단만으론 부족, 요약 + 절단 (부분 요약)
  • compact_only — 줄일 도구 결과 없음(toolResultReducibleChars ≤ 0), 요약이 유일 (요약 필수)

  • run.ts:2707 compactContextEngineWithSafetyTimeout (실제 압축 실행)

④ 요약 품질 감사 — 모델 요약을 믿지 않는다

요약이 불가피할 때도 모델 출력을 그대로 쓰지 않는다 — 생성 → 감사 → 폴백 루프.

  • 구조 강제: 요약은 5개 섹션을 정확한 제목으로 가져야 한다 — ## Decisions / ## Open TODOs / ## Constraints/Rules / ## Pending user asks / ## Exact identifiers. 기본 지시문은 대화 언어 유지 + 코드·경로·식별자·오류 변형 금지. 운영자 커스텀 지시문은 wrapUntrustedInstructionBlock으로 인젝션 방어.
  • 3중 감사: ① 필수 5섹션 존재? ② 정확한 식별자(ID·URL·경로·포트·해시·날짜) 모두 보존? ③ 최신 사용자 요청 반영? → { ok, reasons[] }.
  • 실패 → 폴백: 깨진 요약은 절대 안 싣는다. 유효했던 이전 요약 재사용 또는 5섹션 스켈레톤. 길이도 ~800자(~200토큰)로 제한.
  • compaction-safeguard-quality.ts:234 auditSummaryQuality (재요약 루프, 기본 ON) · :105 buildStructuredFallbackSummary
  • compaction-instructions.ts:23 길이 제한 · REQUIRED_SUMMARY_SECTIONS 섹션 계약

컨텍스트 가지치기 & 엔진

요약/절단과 별개로 토큰 압력 하에 오래된 내용을 다듬는다:

  • 이미지 제거 마커: 무거운 이미지 블록을 [image removed during context pruning]로 치환 — 흔적은 남기고 토큰만 회수.
  • 부트스트랩 안전장치: 첫 사용자 메시지 이전은 절대 가지치지 않는다 — 초기 시스템/부트스트랩 컨텍스트 보호.
  • 가지치기 가능한 도구만: makeToolPrunablePredicate로 안전한 도구 결과만 대상.
  • 엔진 격리-헬스: 상위 context-engine/는 작동 모드(normal·fallback·degraded)를 갖춰, 엔진이 문제를 일으켜도 런타임이 degraded로 살아남는다.

⑤ 재생 안전성 (replay safety)

이미 메시지를 보냈거나·세션을 spawn했거나·cron을 기록한 턴을 그대로 재생하면 중복이 난다. OpenClaw는 재시도 전에 부수효과 증거로 "재생해도 안전한가"를 판정한다.

  • 재생 금지 증거: unsafe 도구(replaySafe≠true) · async 시작된 도구 · 메시지 전송 증거 · 세션 spawn 수락 · cron 기록 성공 > 0 — 하나라도 있으면 재생 금지.
  • fail-closed: 메타데이터가 없는 옛 attempt는 보수적으로 replay-UNSAFE 기본값.
  • 왜 비법인가: 대부분의 재시도 구현은 멱등성을 무시하고 그냥 다시 돌린다 → 중복 전송·중복 실행. OpenClaw는 안전할 때만 재생하고, 안전할 때만 모델을 바꿔 다시 시도한다 — 복구가 새 문제를 만들지 않게 하는 규율.
  • tool-replay-safety.ts:6–39 읽기전용 도구만 재생 허용 · replay-state.ts:60–65 replaySafe = !replayInvalid && !hadPotentialSideEffects
  • delivery-evidence.ts:217–232 전송/스폰/cron 증거 · run.ts:2383–2388 · 3810 · 3826 재시도 게이트

⑥ 도구 호출 복구 — 모든 방언을 받아준다

모델은 도구 호출을 자주 망가뜨린다 — 곡선 따옴표, 잘린 JSON, 앞에 붙은 잡소리, 모델별 직렬화 방언. 스트림을 감싸 파서 수준에서 흡수해, 모델이 문법을 더듬어도 도구가 정상 실행된다.
