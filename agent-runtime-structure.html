OpenClaw 🦞 / 큰 그림: Gateway ↔ Agent Runtime

에이전트 런타임 디렉토리 구조

OpenClaw를 크게 나누면 게이트웨이(컨트롤 플레인)와 에이전트 런타임입니다. 이 문서는 그중 에이전트 런타임을 구성하는 세 영역의 하위 디렉토리를 정리합니다. 런타임 본체는 src/agents/에 있지만, 모델 연결과 공유 코어가 두 곳에 더 걸쳐 있습니다.

“OpenClaw owns the built-in agent runtime directly. The runtime code lives under src/agents/, model/provider helpers live under src/llm/, and reusable core lives in packages/agent-core/.”
— docs/agent-runtime-architecture.md

src/agents/ — 런타임 본체

~1,007 files · 17 subdirs

에이전트 런타임의 심장. 실행 루프·세션·도구·훅·샌드박스가 모두 여기 있습니다. 최상위에는 도구 정의(agent-tools*.ts), 설정(agent-settings.ts), 종료 상태 정규화(agent-run-terminal-outcome.ts) 같은 OpenClaw 소유 파일들이 있고, 아래 하위 폴더로 영역이 나뉩니다.

embedded-agent-runner/151 files ★
내장 에이전트 시도 루프(attempt loop), 프로바이더 스트림 어댑터, 컴팩션, 모델 선택, 세션 연결. 런타임의 실제 실행 엔진. (run/ 하위 포함)
tools/144 files ★
OpenClaw 소유 도구 정의·스키마·정책·before/after 훅 어댑터·호스트 편집 지원. 에이전트가 호출하는 read·exec 등 도구들.
sessions/42 files ★
세션 영속화, 확장 로딩, 리소스 탐색, 스킬·프롬프트·테마, TUI 기반 도구 렌더러.
harness/33 files ★
런타임 하네스 — 라이프사이클, 훅 컨텍스트/히스토리, 컴팩션·복구, 정책, 컨텍스트 엔진 수명주기, builtin-openclaw.ts.
agent-hooks/9 files ★
내장 런타임 훅 — 컴팩션 안전장치, 컨텍스트 가지치기(pruning) 등.
sandbox/76 files
도구/명령의 샌드박스 실행 환경 — 격리·실행 정책.
auth-profiles/76 files
에이전트 인증 프로필 관리 — 자격증명·프로바이더 인증 상태.
cli-runner/31 files
openclaw agent 등 CLI에서 에이전트 실행 진입 경로.

embedded-agent-helpers/18 files

utils/17 files
에이전트 영역 공용 유틸리티.
runtime-plan/10 files

schema/5 files

runtime/3 files
@openclaw/agent-core를 감싸는 OpenClaw 파사드 + 로컬 프록시 유틸 (→ 영역 3 연결점).
modes/interactive/UI

src/llm/ — 모델 · 프로바이더 연결

2 subdirs + 핵심 파일

런타임이 실제 LLM과 통신하기 위한 모델/프로바이더 레지스트리, 전송 헬퍼, 프로바이더별 스트림 구현. 에이전트 본체는 여기를 통해 Anthropic·OpenAI·Google 등으로 요청을 흘려보냅니다.

packages/agent-core/ — 재사용 공유 코어

독립 패키지 · src/

코어와 (잠재적으로) 다른 호스트가 공유하는 하위 수준 에이전트 코어. 하네스 타입·메시지·컴팩션 헬퍼·프롬프트 템플릿·도구/세션 계약을 담습니다. src/agents/runtime/가 이 패키지를 파사드로 감싸 사용합니다.

src/harness/dir ★
